# NetworkSwitcher

`NetworkSwitcher` 是一个面向 Windows 的本地网络路由管理 GUI 工具，适用于多网卡和 OpenVPN 并存的场景，用来在不同网络模式之间快速切换。

## 项目简介

在本地局域网、远程局域网和普通联网环境之间频繁切换时，手工调整网卡、路由和 VPN 进程通常比较繁琐。`NetworkSwitcher` 将这些操作集中到一个桌面程序中，提供可视化配置、模式切换、日志查看和托盘常驻能力。

## 核心功能

| 功能 | 说明 |
|---|---|
| 三种模式切换 | 支持在 `本地局域网`、`远程局域网`、`无规则` 三种模式间切换 |
| 网卡管理 | 支持启用或禁用指定本地网卡 |
| 路由管理 | 支持读取、添加、删除本地 IPv4 路由 |
| 路由表查看 | 可查看当前系统路由表 |
| 路由表清理 | 支持清空本机路由表 |
| OpenVPN 托管 | 支持启动、接管输出并停止由本程序启动的 `openvpn.exe` |
| 日志记录 | 记录应用日志、PowerShell 执行日志和 OpenVPN 输出 |
| 托盘驻留 | 支持托盘图标、托盘菜单和窗口唤起 |
| 开机自启 | 支持通过 Windows 计划任务配置开机启动 |
| Hosts 快捷入口 | 支持快速定位系统 `hosts` 文件 |

## 模式说明

| 模式 | 行为 |
|---|---|
| 本地局域网 | 关闭程序托管的 OpenVPN，启用本地网卡，并将配置的路由写入本地网卡 |
| 远程局域网 | 删除本地局域网相关路由，禁用本地网卡，并启动 OpenVPN |
| 无规则 | 关闭 OpenVPN，移除程序管理的路由，恢复本地网卡，回到系统默认状态 |

## 使用说明

### 1. 启动程序

- 运行 `NetworkSwitcher.exe`
- 程序启动后可显示主界面，并常驻系统托盘

### 2. 配置基础信息

首次使用时需要在设置界面完成以下内容：

- 本地局域网网卡
- 本地局域网需要管理的目标网段（CIDR）
- `openvpn.exe` 路径
- `.ovpn` 配置文件路径
- OpenVPN 额外参数
- 开机自启
- 启动时最小化
- 操作超时

### 3. 切换模式

完成配置后，即可在主界面切换不同网络模式。程序会根据当前模式自动调整路由、网卡状态和 OpenVPN 进程。

### 4. 查看日志与辅助工具

- 在主界面查看应用日志和执行输出
- 在设置或工具界面查看当前路由表
- 使用快捷入口定位系统 `hosts` 文件

## OpenVPN 管理边界

| 项目 | 说明 |
|---|---|
| 管理范围 | 仅管理由本程序启动的 OpenVPN 进程 |
| 启动方式 | 使用可配置的 `openvpn.exe --config ...` 方式启动 |
| 停止方式 | 优先采用受控退出方式，不直接依赖强制结束进程 |

## 构建说明

### 前置条件

- Windows 开发环境
- Qt 5.12.3
- 支持 C++ 的 MSVC 工具链
- qmake

### 构建步骤

```powershell
qmake NetworkSwitcher.pro
nmake
```

如需生成可分发目录，请在构建后根据 Qt 部署要求补齐运行库与插件。

## 项目结构

| 目录 / 文件 | 说明 |
|---|---|
| `app/` | 应用控制逻辑 |
| `core/` | 核心配置与模式逻辑 |
| `models/` | 数据模型 |
| `services/` | 网络、日志、进程、OpenVPN、自启动等服务 |
| `viewmodels/` | 视图模型 |
| `ui/` | QML 界面 |
| `assets/` | 图标与许可证资源 |
| `resources.qrc` | Qt 资源清单 |
| `NetworkSwitcher.pro` | qmake 工程文件 |

## 当前限制

- 仅支持 Windows
- OpenVPN 仅管理由本程序启动的实例
- 程序依赖本机具备相应网络配置权限
